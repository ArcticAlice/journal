# Impeccable Design Context

## Design Context

### Users
Sole user: the developer themselves. Context is daily — opening the app to log tasks and journal entries, then reviewing a month's worth of activity via the summary overlay. The job is fast, private capture followed by occasional reflection. No one else will ever see this interface; it can be fully optimized for one person's taste and workflow.

### Brand Personality
**Moody, bold, austere.**

This is a private instrument, not a consumer product. It should feel like a well-worn personal notebook that happens to be on a screen — atmospheric, high-contrast, slightly severe. No friendliness signals, no onboarding warmth. Just the work.

### Aesthetic Direction
**Dark as the absolute foundation.** Pure black (`#000000` / `#0A0A0A`) is the canvas — not a backdrop but the material itself.

**Two-color tension as the emotional core.**
- Crimson `#750D37` / `#6D1942` — endings, closures, deletion, warning. The color of a closed door.
- Cyan `#00B4D8` — creation, action, progress, selection. The color of something beginning.
- Their gradient (`#750D37` → `#00B4D8`) is the app's signature moment — use it sparingly on structural headers only, not decoratively.

**Typography as texture.** The global 2.5px letter-spacing is the brand's voice — deliberate, spaced-out, slightly cold. Every piece of text should feel weighed before it was placed. Do not reduce or override this.

**Anti-references:**
- No pastel, soft, or muted tones
- No rounded/bubbly UI — avoid pillowy border-radius, iOS-style cards, consumer-app friendliness
- No productivity-SaaS aesthetics (Notion, Linear, Todoist)

### Design Principles

1. **Black is the stage.** Every element is composed against pure black. Use backgrounds only to create depth or distinction — never for decoration. Nothing competes with the dark.

2. **Crimson and cyan carry meaning.** Crimson = stop/close/end. Cyan = create/go/begin. Use each with intent. Never swap them for visual variety. A "save" button is always cyan. A "close/delete" element is always crimson.

3. **Letter-spacing is non-negotiable.** The 2.5px global tracking is the app's typographic identity. It creates the cold, measured tone that makes this feel like a personal instrument. Never override it.

4. **Edge over curve.** Prefer sharp or subtly-rounded corners (≤6px radius). Reserve slightly rounder radii (10–15px) for primary card-level containers only. No pill shapes, no bubble borders.

5. **Bold contrast, no apology.** This is a private space — it can take aesthetic risks. High contrast, saturated color hits, strong visual hierarchy. When in doubt, push harder rather than softer.

### Color Tokens
| Token | Value | Role |
|---|---|---|
| `--bg` | `#000000` | Page background |
| `--surface` | `#0A0A0A` | Card/panel surfaces |
| `--crimson` | `#750D37` | Close, delete, destructive |
| `--crimson-dim` | `#6D1942` | Borders, subtle crimson use |
| `--cyan` | `#00B4D8` | Create, save, interactive |
| `--text-primary` | `#FFFFFF` | Primary text |
| `--text-accent` | `#00B4D8` | Accented/interactive text |
| `--gradient` | `linear-gradient(90deg, #750D37 0%, #00B4D8 100%)` | Header stripe only |
